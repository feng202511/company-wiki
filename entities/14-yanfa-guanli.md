# 研发管理制度

**版本：** V1.0 | **发布：** 2026年6月 | **制定部门：** 量子研发部/软件研发部

## 管理层次
- **PMC（项目管理委员会）** — 最高决策机构，CEO/COO/CTO等组成
- **设计评审委员会** — 技术设计把关
- **技术评审委员会（TRB）** — 技术方案/风险/变更评审

## 产品开发类项目阶段（5.1）

### ① 立项规划阶段
需求分析 → MRD（市场需求文档）→ PRD（产品需求文档）→ 设计方案 → 立项评审

### ② 工程验证阶段（EVT）
工程样机功能/性能初步验证 → 识别设计缺陷 → 工程验证评审

### ③ 设计验证阶段（DVT）
全面功能/性能/稳定性测试 → 固化技术指标与工艺文件 → 设计验证评审

### ④ 生产验证阶段（PVT）
小批量试产 → 验证量产工艺 → 问题闭环 → 生产验证评审

### ⑤ 结项归档
项目总结报告 → 结项评审 → 成果归档 → 技术成果复用

## 内部技术研发类项目阶段（5.2）

### ① 立项阶段
立项建议书 → 可行性分析 → 立项评审（PMC）

### ② 技术实现阶段
方案规划 → 研发实施（含里程碑管理）→ 问题与风险管控

### ③ 指标验证阶段
多维度验证 → 场景化测试 → 指标达标确认

### ④ 结项阶段
结项材料整理 → 结项评审 → 归档

## 项目管理要求（第六章）
整合管理、范围管理、进度管理、预算管理、风险管理、质量管理、沟通管理、知识产权管理
